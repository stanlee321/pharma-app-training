# C-FFI Contract: pk-engine → Swift

## 1. Purpose

This document defines the exact C-FFI boundary that the Rust `pk-engine` crate must expose. The iOS SwiftUI app will consume these functions through a bridging header generated by `cbindgen`.

## 2. Types

All types must be `#[repr(C)]` in Rust.

### 2.1 Input Types

```rust
#[repr(C)]
pub struct PatientParams {
    pub weight: f64,       // kg
    pub height: f64,       // cm
    pub age: f64,          // years
    pub gender: i32,       // 0 = male, 1 = female
    pub dilution: f64,     // mg/ml (drug concentration in syringe)
}

#[repr(C)]
pub struct TargetEvent {
    pub time: f64,         // seconds from simulation start
    pub concentration: f64,// target concentration (mcg/ml or ng/ml, model-dependent)
    pub target_type: i32,  // 0 = plasma, 1 = effect-site
}

#[repr(C)]
pub struct SimulationInput {
    pub model_id: *const c_char,       // null-terminated: "marsh", "schnider", "minto", "hannivoort"
    pub patient: PatientParams,
    pub targets: *const TargetEvent,   // pointer to array
    pub target_count: u32,             // length of targets array
    pub time_range_seconds: f64,       // total simulation duration
    pub resolution_seconds: f64,       // time step between output points
}
```

### 2.2 Output Types

```rust
#[repr(C)]
pub struct CurvePoint {
    pub time: f64,                     // seconds
    pub plasma_concentration: f64,     // Cp (mcg/ml or ng/ml)
    pub effect_concentration: f64,     // Ce (mcg/ml or ng/ml)
    pub amount_v1: f64,               // mg in central compartment
    pub amount_v2: f64,               // mg in rapid peripheral
    pub amount_v3: f64,               // mg in slow peripheral
    pub amount_effect: f64,           // Ce (same as effect_concentration)
    pub infusion_rate: f64,           // mg/min (current infusion rate)
}

#[repr(C)]
pub struct SimulationOutput {
    pub points: *mut CurvePoint,      // heap-allocated array
    pub point_count: u32,             // length of points array

    // Compartment volumes (for animation sizing)
    pub v1_volume: f64,               // ml
    pub v2_volume: f64,               // ml
    pub v3_volume: f64,               // ml

    // Rate constants (for particle animation speed)
    pub k10: f64,                     // 1/min
    pub k12: f64,
    pub k21: f64,
    pub k13: f64,
    pub k31: f64,
    pub ke0: f64,
}

#[repr(C)]
pub struct TCIResult {
    pub bolus_mg: f64,                // Bolus dose in mg
    pub bolus_ml: f64,                // Bolus in ml (= bolus_mg / dilution)
    pub bolus_mcg_per_kg: f64,        // Bolus in mcg/kg
    pub infusion_rate_mg_hr: f64,     // Maintenance infusion in mg/hr
    pub infusion_rate_ml_hr: f64,     // Maintenance infusion in ml/hr
    pub infusion_rate_mcg_kg_hr: f64, // Maintenance infusion in mcg/kg/hr
}

#[repr(C)]
pub struct ValidationRules {
    pub min_weight: f64,
    pub max_weight: f64,
    pub min_height: f64,
    pub max_height: f64,
    pub min_age: f64,
    pub max_age: f64,
    pub requires_height: i32,         // 0 = no, 1 = yes
    pub requires_gender: i32,         // 0 = no, 1 = yes
}

#[repr(C)]
pub struct ModelInfo {
    pub id: *const c_char,            // static string, e.g. "marsh"
    pub drug: *const c_char,          // e.g. "Propofol"
    pub name: *const c_char,          // e.g. "Marsh"
    pub concentration_unit: *const c_char, // "mcg/ml" or "ng/ml"
    pub display_factor: f64,          // multiply internal mg/ml by this for display
                                      // mcg/ml → 1000.0, ng/ml → 1000000.0
}
```

## 3. Exported Functions

### 3.1 Full Simulation

```rust
/// Compute full time-course simulation with one or more target events.
///
/// Returns 0 on success, -1 on invalid model_id, -2 on invalid patient params,
/// -3 on solver failure.
///
/// Caller MUST call `pk_engine_free_output` on the output when done.
#[no_mangle]
pub extern "C" fn pk_engine_simulate(
    input: *const SimulationInput,
    output: *mut SimulationOutput,
) -> i32;
```

### 3.2 TCI Point Query (Fast Path)

```rust
/// Compute bolus and infusion rate to reach a target concentration
/// from the current state. This is the fast path used during drag gestures.
///
/// current_state: pointer to [x1, x2, x3, Ce] (4 doubles)
/// Returns 0 on success, negative on error.
#[no_mangle]
pub extern "C" fn pk_engine_compute_tci(
    model_id: *const c_char,
    patient: *const PatientParams,
    current_state: *const f64,         // [x1, x2, x3, Ce] — 4 elements
    state_len: u32,                    // must be 4
    target_concentration: f64,
    target_type: i32,                  // 0 = plasma, 1 = effect-site
    result: *mut TCIResult,
) -> i32;
```

### 3.3 Memory Management

```rust
/// Free the CurvePoint array allocated by pk_engine_simulate.
/// Must be called exactly once per successful pk_engine_simulate call.
#[no_mangle]
pub extern "C" fn pk_engine_free_output(
    output: *mut SimulationOutput,
);
```

### 3.4 Model Discovery

```rust
/// Get the number of available models.
#[no_mangle]
pub extern "C" fn pk_engine_model_count() -> u32;

/// Get info about model at index (0-based).
/// Returns 0 on success, -1 if index out of bounds.
/// Strings in ModelInfo are static and do NOT need to be freed.
#[no_mangle]
pub extern "C" fn pk_engine_model_info(
    index: u32,
    info: *mut ModelInfo,
) -> i32;

/// Get validation rules for a model.
/// Returns 0 on success, -1 if model_id not found.
#[no_mangle]
pub extern "C" fn pk_engine_get_validation(
    model_id: *const c_char,
    rules: *mut ValidationRules,
) -> i32;
```

## 4. Error Codes

| Code | Meaning |
|------|---------|
| 0 | Success |
| -1 | Unknown model_id |
| -2 | Invalid patient parameters (out of validation range) |
| -3 | ODE solver failure (numerical instability) |
| -4 | Null pointer argument |
| -5 | Invalid state_len (must be 4) |

## 5. Thread Safety

- All functions are **thread-safe** (no global mutable state)
- Each function call creates its own model instance and solver
- Multiple concurrent calls from different Swift threads are safe
- The `SimulationOutput.points` pointer is owned by the caller after `pk_engine_simulate` returns

## 6. Build Outputs

The crate must produce:

1. **`libpk_engine.a`** — static library for `aarch64-apple-ios`
2. **`pk_engine.h`** — C header (generated by cbindgen)

### cbindgen.toml

```toml
language = "C"
include_guard = "PK_ENGINE_H"
autogen_warning = "/* Auto-generated by cbindgen. Do not edit manually. */"
no_includes = true

[export]
include = [
    "PatientParams",
    "TargetEvent",
    "SimulationInput",
    "CurvePoint",
    "SimulationOutput",
    "TCIResult",
    "ValidationRules",
    "ModelInfo",
]

[export.rename]
# Keep Rust names as-is in C

header = """
#include <stdint.h>
#include <stdbool.h>
"""
```

### Build Script

```bash
#!/bin/bash
# build-ios.sh
set -e

# Build for device
cargo build --release --target aarch64-apple-ios

# Build for simulator (Apple Silicon)
cargo build --release --target aarch64-apple-ios-sim

# Generate header
cbindgen --config cbindgen.toml --crate pk-engine --output include/pk_engine.h

echo "Build complete:"
echo "  Device lib:    target/aarch64-apple-ios/release/libpk_engine.a"
echo "  Simulator lib: target/aarch64-apple-ios-sim/release/libpk_engine.a"
echo "  Header:        include/pk_engine.h"
```

## 7. Testing the FFI

```rust
#[cfg(test)]
mod ffi_tests {
    use super::*;

    #[test]
    fn test_simulate_marsh() {
        let input = SimulationInput {
            model_id: b"marsh\0".as_ptr() as *const c_char,
            patient: PatientParams {
                weight: 70.0, height: 170.0, age: 40.0,
                gender: 0, dilution: 10.0,
            },
            targets: &TargetEvent {
                time: 0.0, concentration: 4.0, target_type: 0,
            },
            target_count: 1,
            time_range_seconds: 600.0,
            resolution_seconds: 1.0,
        };
        let mut output = std::mem::zeroed::<SimulationOutput>();

        let status = pk_engine_simulate(&input, &mut output);
        assert_eq!(status, 0);
        assert!(output.point_count > 0);

        // First point should show bolus effect
        let first = unsafe { &*output.points };
        assert!(first.plasma_concentration > 3.0); // near target after bolus

        pk_engine_free_output(&mut output);
    }
}
```
